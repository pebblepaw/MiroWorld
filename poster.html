05 — Screen 3

Social Simulation — OASIS Engine

Analysis questions → seed posts
→
N rounds: read · post · comment · like · repost
→
Checkpoint metrics written per round
  • Engine: OASIS (CAMEL-AI) — open-source multi-agent social interaction framework, Python 3.11 sidecar
  • Each agent reads current feed and LLM-generates its action, conditioned on its Nemotron persona
  • Checkpoint scoring: every agent answers all quantitative analysis questions (1–10 scale) after each round
  • Feed ranking: Reddit-style hot score; controversy boost switch elevates high-engagement controversial posts
  • Live UI: round counter, elapsed time, activity counters, dynamic metric cards, hottest thread, expandable comment threads
  • Backend streams round events via SSE; state persisted in SQLite + AppContext across page navigation

09 — Checkpoint Scoring Pipeline

Score Parsing & Stance Classification

OASIS writes free-text LLM responses into simulation_checkpoints as metric_answers. The backend method _extract_metric_score() parses these into numeric scores.

Input patternParsed valueStance
"7/10", "7/10. I think…" 7.0 Supporter ≥ 7
"Yes" 10.0 Supporter ≥ 7
"6.5" 6.5 Neutral 5–6.9
"No" 1.0 Dissenter < 5
Free text, no leading number None Excluded from scoring
Aggregate vs. Per-Metric Scoring
Aggregate mode
No metric_name specified — opinion flow & polarization use average of all parseable metric scores. Group chat: min score for dissenters, max for supporters.
Per-metric mode
All endpoints score agents using only the specified metric from checkpoint data. MetricSelector dropdown enables this on all metric-aware analytics panels.

13 — Cloud Hosting

AWS Architecture & Cost Management

User ──HTTPS──▶ CloudFront (CDN / HTTPS termination) ~$1/mo │ ├─▶ S3 (React static build) ~$0.02/mo │ └─▶ ALB (Application Load Balancer) ~$16/mo │ ▼ ECS Fargate (FastAPI backend + OASIS sidecar) ~$15–30/mo │ ├─ EFS (SQLite databases + uploaded files) │ └─ RDS PostgreSQL (multi-tenant sessions, FTS memory) ~$15/mo Total estimated: ~$35–50/month · AWS $200 credits → 4–6 months runway

Why ECS Fargate, not Lambda

  • OASIS simulations run minutes, not milliseconds — Lambda's 15-min timeout is incompatible
  • Simulation needs persistent filesystem for SQLite across rounds — Lambda is stateless
  • Fargate runs the existing Docker container as-is, zero refactoring

Scaling & Cost Management

TechniquePurpose
